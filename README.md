# コミットメッセージのコツ　 Prefix をつける

- コミットメッセージの先頭につけとく単語

  - feat← コードを追加した場合（わからなければこれつけとけばいい）
  - docs← ドキュメント管理をした場合
  - refactor← リファクタリングした場合

- [https://qiita.com/konatsu_p/items/dfe199ebe3a7d2010b3e]

# ブランチ戦略のコツ

- ブランチの種類を main,develop,feature に分けます。

  - main ブランチ　 develop がある程度まとまったらマージするブランチ（月 1 ぐらいの頻度）
  - develop ブランチ（一つしかない）　 feature ブランチの修正がある程度まとまったら、マージするブランチ（週 1 ぐらいの頻度）
  - feature ブランチ（複数ある）　目的に合わせた細かい修正ができるブランチ

- gitflow というブランチ戦略
- [https://zenn.dev/dannykitadani/articles/15e52a5a40a0bf]
